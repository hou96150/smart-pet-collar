## 第一版完成範圍

- 音訊門檻觸發與低於門檻一段時間後停止的狀態機。
- `librosa` Mel 頻譜產生功能與可替換的 TensorFlow 模型介面。
- Windows 不需安裝 GPIO 或 TensorFlow 即可啟動的模擬模式。
- Python CLI 將辨識事件送到 Node.js 後端。
- 後端依情緒、信心與暫定規則計算低、中、高風險。
- SSE 即時更新網站，並以 JSONL 持久保存歷史事件。
- 高風險網站警告與安全蜂鳴器 mock。
- 手機與 Raspberry Pi 位於同一個 Wi-Fi 時，可透過區域網路 IP 開啟網站。

## 已知限制

1. 目前沒有原始音訊資料集、類別完整清單、訓練程式與模型權重。
2. 第一版的情緒輸出是明確標示的模擬事件，不可宣稱為真實 AI 辨識成果或準確率。
3. 風險規則是為串接展示建立的暫定值，不是經研究驗證的安全標準。
4. 實體 Raspberry Pi、麥克風、電池續航、項圈重量與 GPIO 蜂鳴器尚未在本次重建中測試。
5. 網站目前服務於區域網路，尚未提供帳號、雲端部署或遠端通知。
